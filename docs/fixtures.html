Fixture corpus

The data-driven conformance corpus. Each case shows its HAMON surface, the expected semantics, a realized score (music21 → Verovio) where the harmony is engravable, and the same label in every source encoding.

How to read this page. Everything that is HAMON — the surface, the per-format exports and the loss numbers (xencoding) — is text, computed directly by the Python reference hamonpy (HAMON → format → HAMON). It never passes through a renderer. The engraved score is illustrative only: music21 realizes the harmony into pitches and Verovio (loaded from CDN) paints it, purely for visual reference. When a score can't be realized/rendered it is simply omitted — that is a display limitation, not a HAMON loss, and it does not affect any measurement here.
